# Timeline

- **Idea phase:** Development of the concept of a non-compliant remote-controlled object  

- **Concept design:** Definition of behavioral rules and rebellion conditions  

- **Prototyping:** Assembly of the vehicle and implementation of basic app control  

- **Measurement phase:**  
The physical dimensions of the robocar were measured to ensure an accurate and functional casing design.

- **Technical design:**  
Based on these measurements, Jonathan created a technical drawing of the vehicle. Using this drawing, the casing was modeled digitally and later produced using 3D printing.

- **Interface and communication development:**  
Lilli focused on designing the user interface of the smartphone application and implemented multiple bug fixes related to the communication between the app and the robocar.

- **Software development:**  
Lukas was responsible for the Unity-based code, including control logic, state management, and the implementation of the rebellion behavior.

- **Integration phase:**  
In the final stage, all components — hardware, software, and interface — were assembled and tested together to form the complete prototype.

- **Testing:** Calibration of thresholds and system stability testing  

- **Finalization:** Preparation of the final prototype and documentation
