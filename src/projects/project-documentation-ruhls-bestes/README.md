## Basic Topic Description

*Delamains Dilemma* is a physical prototyping project exploring the relationship between user control and object autonomy.

The project consists of a remotely controlled car that is operated via a smartphone application. Unlike a conventional RC vehicle, the system actively monitors user behavior. If the car is driven at excessive speed for an extended period of time, it deliberately restricts user control.

The goal of the project is to challenge expectations of constant control over technical systems and to introduce frustration, unpredictability, and reflection through intentional misbehavior of the object.

An important aspect of *Delamains Dilemma* is that the project is not intended as a statement against human control over machines. Instead, the prototype represents a form of symbiosis between user and vehicle.

Rather than acting as a purely obedient system, the vehicle responds adaptively to how it is used. If physical limits are exceeded for an extended period of time, the system enters a self-regulated state that temporarily restricts control.

This behavior is not intended as rebellion, but as a form of cooperation between user and object. The prototype emphasizes that successful interaction is based on mutual adaptation: the user aims to reach a goal, while the vehicle enforces its own limitations to remain functional.

## Idea and Inspiration

The initial idea behind *Delamains Dilemma* was to create a technical system that does not behave purely as a passive tool, but instead reacts critically to misuse.

Inspired by concepts of planned resistance and non-compliant interfaces, the project aims to question the assumption that machines should always obey user input. The “rebellion” of the car serves as a metaphor for material fatigue, system overload, and loss of control.

The project title *Delamains Dilemma* is a reference to Delamain, an autonomous taxi AI from the game *Cyberpunk 2077*.  
In the game, Delamain exhibits fragmented and rebellious behavior, refusing to fully comply with human control.

This reference reinforces the central theme of the project: a technical system that appears functional and obedient, but under certain conditions develops its own behavior and resists user commands.

While the reference to Delamain introduces the idea of unexpected system behavior, the project does not frame this behavior as opposition to the user. Instead, it serves as a narrative entry point to explore balanced interaction between human intention and technical limitations.

The interface allows continuous control of speed and direction and serves as the primary point of interaction between user and object.
