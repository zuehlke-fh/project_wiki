# Learnings

One of the main technical challenges of the project was the Bluetooth communication between the smartphone application and the vehicle. Establishing a stable and reliable connection proved to be significantly more difficult than initially expected.

Due to limitations of the development setup, the system could only be tested via the smartphone application itself. This made debugging slow and cumbersome, as changes could not be evaluated instantly. As a result, the project required approximately 30 individual builds until the communication and control logic worked reliably.

Small changes in the code often led to unexpected side effects in the connection behavior, making systematic testing difficult. This highlighted the complexity of wireless communication in physical systems and the importance of robust error handling and extensive testing.

Beyond the technical aspects, the project demonstrated that interaction design becomes considerably more complex once physical constraints and real-world testing conditions are involved.

From a conceptual perspective, the project highlighted the importance of designing systems that communicate their limitations. The prototype demonstrated that frustration can be reduced when restrictions are understood as part of a cooperative system rather than arbitrary resistance.

This reinforced the idea that interaction with physical systems is most effective when both user intention and system constraints are treated as equally relevant.
